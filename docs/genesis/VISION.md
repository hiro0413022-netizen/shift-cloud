# VISION — YOZAN GENESIS

> 一文の定義
> **YOZAN GENESISとは、古川博庸の経営脳をAI化し、GOLF WING・FRUNK GOLF・KALLINOS・YOZAN全体を24時間自律的に動かす、AI社員型の企業OSである。**

作りたいのは「便利な業務システム」ではなく、**古川さんの分身（CEO AI）を中心に会社そのものを自動成長させる仕組み**。
古川さんは「俺が考える」だけ。AIが形にし、現場が回り、数字が見え、改善が勝手に走り、会社が増殖する。

このドキュメントは正典（North Star）。設計判断は `DECISIONS.md`、実作業は `NEXT_TASKS.md` に落とす。

---

## 1. 完成した状態

朝、Genesis Cockpitを開くと CEO AI が報告する：

- 今日のYOZAN全体スコアは82点
- GOLF WING は体験予約が目標より2件不足
- FRUNK GOLF はオープン準備タスクが3件遅れ
- KALLINOS は在庫回転が悪い商品あり
- 今週の売上予測は目標比92%
- **今日、古川さんが判断すべきことは3つ**

細かい作業は古川さんがやらない。AIがシフト・勤怠・在庫を見て、発注を提案し、SNS/LINE/求人文を作り、スタッフのタスクを整理し、GitHub Issue〜PRまで作り、売上・人件費・会員・退会リスクを見て打ち手を出す。
古川さんは最後に **「承認する / 方向性を変える / GOを出す」** だけ。

---

## 2. 開発順序（本丸から）

全事業を同時に完璧にしない。型を1つ作り、横展開する。

1. **第1本丸：GOLF WING運営OS ＋ CEO AI Command Center**
   すでに現場・売上・会員・スタッフ・在庫・レッスン・発注・勤怠・予約が揃う＝一番リアルな実験台。
   統合対象：売上 / 会員 / 体験予約 / 入会率 / 退会率 / 勤怠・シフト / 人件費 / 在庫 / 発注 / レッスン / SNS / タスク / AI改善提案 — **これを1つのCockpitに統合**。
2. **FRUNK GOLF OS**（姫路新店）：オープン準備、会員獲得、レッスン、法人会員、バー運営、売上予測。
3. **KALLINOS Brand OS**：商品企画、在庫、販売、SNS、世界観、EC、顧客管理。
4. **YOZAN Corporate OS**：全事業の売上・人材・資金・タスク・開発・営業・マーケを統合。
5. **YOZAN GENESIS 販売モデル**：他のインドアゴルフ施設に売れる業務OSへ。＝YOZAN自身の商品になる。

## 8. 開発の完成形（データの流れ）

1. Genesis Kernel を中枢に置く（全AI・全システムを接続）
2. 各モジュールを接続：Shift Cloud / 勤怠 / 在庫 / 発注 / 売上 / 会員 / レッスン / SNS / 採用 / 資料 / GitHub開発
3. CEO AI がすべてを読む
4. 循環：**データが集まる → AIが分析 → 改善案 → 必要なAIにタスクを振る → 実行できるものは実行 → 危険なものだけ古川さんに承認要求**

## 9. 最終ゴール

**古川さんが現場作業から抜けても、会社が成長し続ける状態。**
新しい事業を思いついた瞬間、CEO AIが事業計画を作り、開発AIがシステムを作り、営業AIが売り方を考え、SNS AIが集客し、経理AIが収支を見て、現場AIが運営まで落とし込む。

俺が考える → AIが形にする → 現場が回る → 数字が見える → 改善が勝手に走る → 会社が増殖する。
