# SAAS_PLAN — GENESISを他社に売るために必要なこと

作成: 2026-07-13。VISION §2-5「YOZAN GENESIS 販売モデル」の実行計画。
古川さんの懸念＝**「売れたときに初期設定が複雑すぎる」**を中心に組み立てる。

---

## 0. 結論（先に要点）

1. 売り物は「ソフト」ではなく**「うちと同じ運営が明日からできる状態」**。だから商品の中心は**オンボーディング**
2. 初期設定の複雑さは (a) セットアップウィザード (b) **AI設定コンシェルジュ（チャットで設定）** (c) 導入代行メニュー の3段構えで潰す
3. 技術的な土台（マルチテナント）は既にある程度できている（company_id分離・RLS）。足りないのは**「自分たちで触らせない」ための道具**

## 1. 今できていること（売る側の資産）

- 全テーブルが company_id でテナント分離済み（RLS標準・DECISIONS #1系）
- モジュール構成: シフト勤怠給与 / 体験受付 / 予約 / 経理 / 法務 / アンケート / キャディ / 資料室 / 社内連絡
- 給与計算は実際の明細と1円一致で検証済み＝**インドアゴルフ運営の実データで鍛えた業務ロジック**が最大の差別化
- AI: 日次レポート・判断リスト・OCR・契約書読取り（Claude API）
- 現場マニュアル（RUNBOOK→/manual配信）の型

## 2. 足りないもの（SaaS化の必須項目）

### A. テナント発行と初期設定（懸念の本丸）

| 項目 | 内容 | 優先 |
|---|---|---|
| テナント発行ウィザード | 会社名・店舗・スタッフ・時給/月給・営業時間を画面の質問に答えるだけで投入。CSV一括取込（スタッフ名簿・会員名簿） | ★★★ |
| **AI設定コンシェルジュ** | チャットで「レッスンの単価を5,000円にして」「打席を2つ追加」→ AIが設定テーブルを更新（危険な変更は確認画面を挟む）。**相手はシステム編集ができない前提なので、これが実質の管理画面になる** | ★★★ |
| デモデータ | 契約前に触れるサンプル店舗（体験→入会→給与まで一巡） | ★★ |
| セルフサービス設定画面 | 単価・手当・締め日・権限のGUI（今はSQL/Claude経由の部分を画面化） | ★★ |

### B. HP・予約サイトの引継ぎ（移行サービス）

- **方針: 「引っ越しは商品に含める」**。先方作業ゼロが理想
- 予約: Smart Hello等の既存予約システムは (1) 当面共存（リンク集に載せるだけ＝GOLF WING方式） (2) Reserve OSへ段階移行 の2択を用意。CSVエクスポートがある予約システムは取込アダプタを作る（Smart Hello取込の設計＝member-os/SMART_HELLO_IMPORT.mdが雛形）
- HP: 既存HPはそのまま・GENESIS群へのリンクを置くだけから始める（corporate siteの型を流用してHP制作をオプション販売してもよい）
- ドメイン・メール: お名前.com運用の知見（yozan-mail-setup）をそのまま手順書化

### C. 商売の土台

| 項目 | 内容 |
|---|---|
| 課金 | Stripe（月額/店舗数課金）。請求書払いも法人向けに用意 |
| 契約・規約 | 利用規約・個人情報の委託（会員名簿を預かる）・SLA。Legal OSで自社管理 |
| インフラ分離 | 顧客ごとにSupabaseプロジェクト分離（推奨・事故時の爆風半径を限定）or 同居＋RLS。**リポジトリは顧客ごとにPrivate化が必須**（現Publicのままは不可） |
| 環境自動構築 | 新規テナント=Supabase+Vercel+env をスクリプト一発で（scaffoldの延長） |
| 監視・サポート | System Networkの死活監視を顧客横断ダッシュボードに拡張。問い合わせはCEO Inboxの型を転用 |
| 運用体制 | 導入担当（現状=古川さん）。1社目はファイン福原氏のSales OS流通網でPGA NOTE顧客に売る導線が現実的 |

### D. AI設定コンシェルジュの実現方法（技術メモ）

- 既にClaude APIをcronで使用中 → チャットUI（/concierge）＋ツール定義（設定の読取り・更新・確認）を追加するだけで骨格はできる
- 権限の線引きはVISION §7と同じ: **AIは提案と下書きまで、実行前に確認画面**（例:「時給を1,200円→1,300円に変更します。よろしいですか？」）
- 変更はすべてaudit_logsに記録（誰がAI経由で何を変えたか）
- 月あたりのAPIコストは従量。料金プランに織り込む

## 3. 進め方（フェーズ）

1. **Phase S0（社内で完成させる）**: FRANK GOLF姫路を「2店舗目のテナント」として発行し、ウィザードで立ち上げてみる＝自社が最初の顧客
2. **Phase S1（設計）**: テナント発行スクリプト＋リポジトリPrivate分離＋Stripe
3. **Phase S2（1社目）**: 知り合いのインドアゴルフ1社に**導入代行つき**で有償トライアル（価格は安く・フィードバック優先）
4. **Phase S3（型化）**: S2の学びをウィザードとAIコンシェルジュに落とす → 営業はSales OS/福原氏ルート

## 4. 価格の考え方（たたき台）

- 初期費用: 導入代行（データ移行・スタッフ研修・マニュアル差し替え）を含めて設定
- 月額: 基本料＋店舗数。AI機能（日次レポート・コンシェルジュ）は上位プラン
- 比較対象: シフト管理SaaS＋予約SaaS＋会計freee等を別々に払うより安い、が目安
