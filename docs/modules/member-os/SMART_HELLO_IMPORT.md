# Smart Hello 取込仕様（GOLF WING 会員系KPI）

DECISIONS #22。GOLF WING会員管理は Smart Hello（株式会社システムディ、公開APIなし）。
会員系KPIは Smart Hello の **Excel/CSVエクスポート → Genesis取込** で接続する（財務 #21 と同型「エクスポートが唯一の真実」）。

このドキュメントは実サンプル2本（2026-07-06取得）に基づく取込スキーマ設計。

## 1. 入力ファイル（2種）

### A. 会員名簿（`会員名簿_YYYYMMDDhhmmss.xlsx`）
- 1シート「会員名簿」、1行目ヘッダ、87列。サンプル=244レコード。
- 1レコード＝1会員（スタッフも会員種類「スタッフ」として混在）。
- **取込に使う主な列**：会員番号 / 氏名 / カナ名 / 性別 / 生年月日 / 年齢 / 入会日 / 退会日 / 退会理由 / 会員種類名 / クラス名 / 所属店舗コード・名 / 休会開始年月 / 休会終了年月 / キャンペーン名 / 支払方法 / 月間来場回数 / 最終来場日。
- ※住所・電話・メール・口座・クレジットカード列等の個人情報はKPIに不要。取込時は**取り込まない or 別テーブルで最小限＋RLS保護**（#3/#11準拠）。

### B. 予約一覧（`予約一覧_YYYYMMDD-YYYYMMDD_....xlsx`）
- 1シート「予約一覧」、1行目ヘッダ、52列。サンプル=2,189行（2026/06分）。期間指定エクスポート。
- 1レコード＝1予約。**主な列**：店舗コード・名 / プログラム種別名 / プログラム名 / 実施場所名（打席）/ 担当スタッフ名 / レッスン日 / 開始・終了時刻 / 予約番号 / **予約区分**（予約 / 体験）/ 予約者会員番号 / 予約者氏名・カナ・生年月日・年齢・性別 / 予約登録日時 / 予約状態 / キャンセル登録日時 / 出欠 / 料金・チケット / 税込金額。
- 注意：今回サンプルは予約状態が全て「予約」、出欠が全て空。キャンセル・出欠を追うには該当列が埋まるエクスポート条件の確認が必要。

## 2. KPI定義（列→算出）

| KPI | 算出ロジック | ソース |
|---|---|---|
| 会員数（在籍） | `退会日` 空 かつ `会員種類名`≠スタッフ の件数 | 会員名簿 |
| 会員構成 | 在籍を `会員種類名` で集計 | 会員名簿 |
| 月次入会数 | `入会日` の年月で集計 | 会員名簿 |
| 月次退会数 | `退会日` の年月で集計（未来日＝退会予定として別集計） | 会員名簿 |
| 退会率（月次） | 当月退会数 ÷ 前月末在籍数 | 会員名簿 |
| 休会数 | `休会開始年月`あり かつ `休会終了年月`空 | 会員名簿 |
| 体験予約数 | `予約区分`=体験 の件数（期間内） | 予約一覧 |
| **入会率（体験→入会）** | 体験者（区分=体験）の **氏名＋生年月日** を会員名簿の入会者に突合し、一定期間内に入会した割合。※体験者は入会まで会員番号を持たないため会員番号では突合不可 | 両ファイル突合 |
| 稼働（予約数） | プログラム種別/打席/日で集計。総枠との比で稼働率 | 予約一覧 |
| 来場活性 | `最終来場日`・`月間来場回数`（退会予兆の早期指標にも） | 会員名簿 |

## 3. 退会予兆（CEO AIアラート候補）
- `退会日` が未来日＝**退会予定**。今すぐ引き止めアクション対象としてアラート化。
- 在籍だが `最終来場日` が長期空白／`月間来場回数`=0 が続く会員＝離脱リスク。

## 4. 取込方式（設計方針）
1. Genesisに会員取込画面（financeの取込#21と同UI思想）。xlsx/CSVをアップロード。
2. 会員名簿＝**全件洗い替え（スナップショット）**方式。取込日時を持つ最新スナップショットを正とする（会員数・退会予定は「今の状態」なので差分ではなくスナップショットが自然）。
3. 予約一覧＝**期間追記**方式。予約番号でユニーク化し重複取込を防止。
4. 取込時に体験→入会の突合（氏名＋生年月日、正規化して照合）を実行し入会率を算出。
5. 集計結果を Kernel の KPI（会員数/入会/退会/退会率/体験予約/入会率）へ反映 → Cockpit表示。労務KPI（#0008）と並置。

## 5. 未決事項（要ユーザー確認）
- 会員数の定義：スタッフ・モニター・トライアルを「会員数」に含めるか（本仕様は在籍かつスタッフ除外を既定）。
- 休会会員を「在籍会員数」に含めるか。
- 予約一覧で **キャンセル・出欠** を取るためのエクスポート条件（今回サンプルは未記録）。
- エクスポート頻度（週次／月次）と担当。
