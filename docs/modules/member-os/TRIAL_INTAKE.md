# 体験予約受付システム 設計（GOLF WING）

## 目的
GOLF WINGの体験予約受付は現在「紙にお客様が手書き → スタッフがExcelに手入力」で非効率。
これを Genesis 内の **Web受付システム**に置き換える。
- お客様：来店時に**タブレットを渡して必要事項を自己入力**（＋同意・電子サイン）。
- スタッフ：予約枠の作成・当日の受付・体験結果・入会可否を**Web入力**。
- 貯まったデータで 0010 の会員系KPI（体験予約数・入会率）を**自動集計**（手動更新を廃止）。

※項目は添付Smart HelloエクスポートのExcel列を参考（データ自体は古いため項目のみ流用）。

## フロー
1. **予約作成**（スタッフ or HP）：体験メニュー・希望日時→確定枠・打席・担当スタッフ・予約経路 を登録。予約番号を自動発行。status=予約。
2. **来店・自己入力**（タブレット）：予約に紐づくトークンURLを開き、お客様が個人情報＋アンケートを入力→同意→電子サイン。status=来店。
3. **体験実施**：出欠（来店/キャンセル/no-show）と体験結果メモを記録。
4. **入会可否**：入会＝会員化（会員化フラグ・入会日）→ 入会率にカウント。見送り＝理由記録（フォロー対象）。

## データモデル（migration 0011想定・追加のみ）
既存標準準拠：company_id + RLSテナント分離、書き込みservice_role+requireActor、論理削除、updated_atトリガー（DECISIONS #11/#17）。

### `mbr_trial_bookings`（体験予約）
company_id / store_id / 予約番号(自動) / 体験メニュー(program) / 予約日時(希望・確定) / レッスン日 / 開始時刻 / 終了時刻 / 打席(実施場所) / 担当スタッフ / 予約経路(HP/電話/来店/紹介/SNS) / status(予約/来店/キャンセル/no-show) / 出欠 / 体験結果メモ / 入会フラグ / 入会日 / guest_id(FK) / created_by / timestamps / deleted_at

### `mbr_trial_guests`（体験者＝見込み客の個人情報・最小限＋RLS保護）
氏名 / カナ / 生年月日 / 性別 / 郵便番号 / 都道府県 / 市区町村以下 / 建物 / 電話番号 / 携帯電話 / メール / DM・連絡可否 / 紹介者 / アンケート(JSONB) / 同意日時 / 電子サイン(画像or座標) / created_at
→ 個人情報テーブルは給与系(#3)同様にRLSでも保護。KPI集計は非PIIの集計値のみ参照。

### タブレット自己入力の認証
予約ごとの**受付トークン**（sha256ハッシュ保存、DECISIONS #12キオスクと同方式）。スタッフが「タブレット受付」を押すと当該予約の入力URLを表示。HQ認証不要の顧客向けルート。

## フォーム項目

### お客様がタブレットで入力
必須：氏名 / フリガナ / 生年月日 / 性別 / 携帯電話番号 / メールアドレス / 郵便番号 / 住所（都道府県・市区町村以下・建物）
アンケート（任意・入会率改善に有用）：ゴルフ経験（未経験/初心者/経験者）/ 来店のきっかけ（HP・SNS・チラシ・紹介・通りがかり 等）/ 体験の目的（上達・健康・コンペ対策 等）/ 現在の課題
同意：個人情報取扱い同意（必須チェック）/ 電子サイン

### スタッフが入力
体験メニュー / 希望日時→確定枠 / 打席 / 担当スタッフ / 予約経路 / 備考 / 出欠 / 体験結果 / 入会可否・入会日 / （見送り時）理由

## KPI連携（0010を自動化）
- `trial_bookings`（体験予約数・当月）= 当月 status≠キャンセル の体験予約件数。
- `conversion_rate`（入会率）= 入会フラグ数 ÷ 体験実施(来店)数。※体験者→入会は本システム内で紐づくため、Smart Hello突合（氏名+生年月日）は不要になる。
- 集計関数を `refresh_shift_cloud_kpis` と同様に用意し、日次レポート時＋手動更新で再集計。

## 段階リリース案
- **Phase 1a（本丸・最小）**：スタッフ用の体験予約 登録＋一覧＋当日ステータス更新（Excel置換）。KPI自動連携（体験予約数・入会率）。← まずここで紙・Excelを廃止できる。
- **Phase 1b**：タブレット自己入力フォーム＋同意・電子サイン（トークン方式）。
- **Phase 2**：見送り客のフォロー（LINE/メール下書きをAI生成）、体験→入会の予兆分析。

## スコープ確定（2026-07-06, DECISIONS #23/#24）
- 体験受付は **1a＋1b を一気に構築**（スタッフWeb入力＋タブレット自己入力＋電子サイン）。
- 次段で **予約サイト/予約システム** を作り、本受付と紐づけて動かす。
- **予約システムの棲み分け**: 当面 GOLF WING(宝塚) は Smart Hello 継続。**姫路 FRANK GOLF は本予約システムを導入**。→ member-os はマルチストア前提で設計（company_id/store_id 分離済み・migration 0011）。
- **会員名簿本体も将来 Genesis に移管**（入会後の会員管理・会費・来場）。mbr_guests はその基盤として設計。
