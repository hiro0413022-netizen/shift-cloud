# Caddy OS — キャディ派遣事業の管理システム（正典）

DECISIONS #45 / migration `0036_caddy_os.sql` / アプリ `apps/caddy-os`（別Vercel・DB共有・port 3005）

## 1. 何を解決するのか

キャディ派遣は **月1つのExcel（CADDIES 2026_N.xlsx）** で運用されていた。移行時に見つかった構造的な問題:

| # | Excelの問題 | Caddy OSでの解決 |
|---|---|---|
| 1 | 委託料シートのNo採番が前月のままコピペ（6月ファイルに `2026-5-001`） | `seq` を自動採番。人は触らない |
| 2 | 売上シートと委託料シートが別 → 同じ派遣の売上と原価が突き合わせられない | **1派遣=1行**（`cad_dispatches`）に売上と原価を両方持つ → 行単位で粗利が出る |
| 3 | 社員（林さん）の交通費が委託料シートに入り、給与でも支給 → **二重計上** | `partner_id`（外注）と `staff_id`（社員）を排他に。社員行は原価0（CHECK制約で強制） |
| 4 | 委託先マスタに無い人が台帳に出る（日高真里子さん） | マスタ参照（FK）。無ければ登録が必要になる |
| 5 | 締め日・振込日がバラバラで入金消込ができない | 取引先マスタに締め日・振込日を保持 |
| 6 | 収支シートが空 → 事業の採算が誰にも見えない | ダッシュボードで月次の売上/外注費/粗利/人工を常時表示 |

## 2. データモデル

```
cad_clients   取引先（ゴルフ場）  code / name / unit_price / closing_day / payment_day / has_contract
cad_partners  委託先（キャディ）  code / name / default_fee / default_transport / main_course
cad_dispatches 派遣（1行=1人1回）  dispatch_date / client_id + sales_amount / (partner_id | staff_id) + fee/transport/special
```

**個人情報は持たない**: 委託先の住所・生年月日・銀行口座はExcelにあるが `cad_partners` には入れていない。
必要になった時点でVault相当の保護（#26）とセットで設計する（SECURITY.md）。

## 3. 原価のルール（最重要）

```
外注費 = Σ (fee_amount + transport_amount + special_amount)   ただし partner_id がある行のみ
社員の派遣 = 原価0                                             人件費は給与システム（#44）が計上
```

DB側の CHECK 制約 `cad_staff_has_no_fee` が、社員行に委託料・特別手当が入るのを**構造で禁止**している。

## 4. 財務への接続

`refresh_caddy_finance(company_id, month)` が `cad_dispatches` → `fin_entries` へ upsert する（source=`caddy_os`）。

- 売上高（sales）= Σ sales_amount
- 外注費（outsourcing）= 上記の外注費
- **人件費は触らない**（給与側の担当。二重計上の防止）

派遣の登録・削除のたびに自動実行される。Genesisの事業別PL・5大KPI（月次売上・営業利益）はこれを読む。

## 6. 画面
