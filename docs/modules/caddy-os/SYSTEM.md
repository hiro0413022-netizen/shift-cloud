# Caddy OS — キャディ派遣事業の管理システム（正典）

DECISIONS #45 / migration `0036_caddy_os.sql` / アプリ `apps/caddy-os`（別Vercel・DB共有・port 3005）

## 1. 何を解決するのか

キャディ派遣は **月1つのExcel（CADDIES 2026_N.xlsx）** で運用されていた。移行時に見つかった構造的な問題:

| # | Excelの問題 | Caddy OSでの解決 |
|---|---|---|
| 1 | 委託料シートのNo採番が前月のままコピペ（6月ファイルに `2026-5-001`） | `seq` を自動採番。人は触らない |
| 2 | 売上シートと委託料シートが別 → 同じ派遣の売上と原価が突き合わせられない | **1派遣=1行**（`cad_dispatches`）に売上と原価を両方持つ → 行単位で粗利が出る |
| 3 | 社員（林さん）の交通費が委託料シートに入り、給与でも支給 → **二重計上** | `partner_id`（外注）と `staff_id`（社員）を排他に。社員行は原価0（CHECK制約で強制） |
| 4 | 委託先マスタに無い人が台帳に出る（日高真里子さん） | マスタ参照（FK）。無ければ登録が必要になる |
| 5 | 締め日・振込日がバラバラで入金消込ができない | 取引先マスタに締め日・振込日を保持 |
| 6 | 収支シートが空 → 事業の採算が誰にも見えない | ダッシュボードで月次の売上/外注費/粗利/人工を常時表示 |

## 2. データモデル

```
cad_clients   取引先（ゴルフ場）  code / name / unit_price / closing_day / payment_day / has_contract
cad_partners  委託先（キャディ）  code / name / default_fee / default_transport / main_course
cad_dispatches 派遣（1行=1人1回）  dispatch_date / client_id + sales_amount / (partner_id | staff_id) + fee/transport/special
```

**個人情報は持たない**: 委託先の住所・生年月日・銀行口座はExcelにあるが `cad_partners` には入れていない。
必要になった時点でVault相当の保護（#26）とセットで設計する（SECURITY.md）。

## 3. 原価のルール（最重要）

```
外注費 = Σ (fee_amount + transport_amount + special_amount)   ただし partner_id がある行のみ
社員の派遣 = 原価0                                             人件費は給与システム（#44）が計上
```

DB側の CHECK 制約 `cad_staff_has_no_fee` が、社員行に委託料・特別手当が入るのを**構造で禁止**している。

## 4. 財務への接続

`refresh_caddy_finance(company_id, month)` が `cad_dispatches` → `fin_entries` へ upsert する（source=`caddy_os`）。

- 売上高（sales）= Σ sales_amount
- 外注費（outsourcing）= 上記の外注費
- **人件費は触らない**（給与側の担当。二重計上の防止）

派遣の登録・削除のたびに自動実行される。Genesisの事業別PL・5大KPI（月次売上・営業利益）はこれを読む。

## 6. 画面

- `/` ダッシュボード — 月次KPI（売上/外注費/粗利/人工）、取引先別、委託先別、6ヶ月推移、財務へ再集計
- `/dispatches` 派遣台帳 — 一覧＋**スプレッドシート風の一括入力**（#46）＋**ゴルフウィング勤務（時給）入力**（#62）
- `/invoices` 請求（受取）— 取引先別の請求サマリ → `/invoices/[clientId]` で請求書（印刷/PDF保存）＋発行記録・入金消込
- `/invoices/payable` 支払 — 委託先別の支払サマリ → `/invoices/payable/[partnerId]` で**キャディ→YOZAN請求書**（#62）＋振込用CSV
- `/availability` 出勤可否 — 委託先×日のカレンダー（○△×・クリックで即保存）
- `/masters` 設定 — 取引先・委託先マスタの**編集**、委託料（ゴルフ場ごと）・交通費単価表（キャディ×ゴルフ場）、台帳表示フラグ・時給（#62）

## 7. 一括入力（#46）

派遣は月40〜60件。1件ずつのフォームは運用に耐えないため、グリッド入力にした。

- **Enter** で次の行（**日付と取引先を引き継ぐ** — 同じ日・同じコースに複数キャディという実運用に合わせた）
- 取引先を選ぶと**単価が自動**、委託先を選ぶと**委託料・交通費が自動**
- 自社スタッフの行は委託料・手当を**入力できない**（給与との二重計上防止）
- 登録後、`renumber_caddy_seq()` が採番を日付順に振り直す（Excelの採番コピペ事故の再発防止）

## 8. 請求書（#46）

実物（2026年6月・加古川ゴルフ倶楽部）を再現。**派遣台帳から自動生成**する。

```
明細  = 「キャディ業務料 (YYYY年 M月D日 分）」 × その日の人工数 × 単価
小計  = Σ明細（税抜）
税額  = floor(小計 × 10%)
合計  = 小計 + 税額（外税）
締切日 = 取引先マスタの締め日（月末 / 20日。全角表記も吸収）
```

- 差出人・振込先は `companies.settings.invoice`（migration 0037で投入）
- 画面表示 → ブラウザの印刷でPDF保存（`@media print` でA4）
- 発行履歴 `cad_invoices` テーブルは用意済み（snapshot保存用。現在は都度生成）
- `tests/caddy-invoice.test.ts` が実物と1円一致で固定

## 9. 出勤可否（#46・2ヶ月後に運用開始予定）

`cad_availability`: 委託先 × 日 × ○（可）/△（要相談）/×（不可）。空欄=未回答。

- セルをクリックするたびに ○→△→×→空欄 と切り替わり、その場で保存
- 派遣が既に入っている日は青枠で表示（重複配置の防止）
- 最下段に「その日に出られるキャディの人数」

**なぜ Shift Cloud ではなく Caddy OS に置くのか**: 委託先は社員ではない。Shift Cloud に載せると
勤怠→給与→人件費として計上され、**委託料との二重計上**になる（0036 が構造で禁止したのと同じ事故）。
自社スタッフ（林さん）のシフト・勤怠は従来どおり Shift Cloud が持つ。

## 10. フェーズ3 — 支払請求書・単価表・時給（#62 / migration 0073）

ユーザー決定（2026-07-24）を反映して4機能を追加。

**① キャディ → YOZAN の請求書（支払）**
委託先が YOZAN へ上げる請求書を `/invoices/payable` で生成。中身は当月の派遣（委託料＋交通費＋特別手当）＋ゴルフウィング勤務（⑤）。
**消費税なし（免税事業者）**で合計する。`buildPayable()`（`lib/invoice.ts`・純粋関数）。

**② 交通費を「キャディ × ゴルフ場」で設定**
`cad_transport_rates`（client_id × partner_id → amount）。派遣入力で自動反映。表になければ委託先の `default_transport` にフォールバック。

**③ 委託料を「ゴルフ場ごと（全キャディ共通）」で設定**
`cad_clients.partner_fee`。派遣入力で自動反映（`partner_fee` → 無ければ委託先 `default_fee`）。派遣ごとの上書きは可。

**④ 台帳に出すキャディを絞り込む**
`cad_partners.show_in_picker`。設定画面のトグルでON/OFF。`getMasters()` は `show_in_picker=true` のみプルダウンに返す（退職・休眠を隠す）。

**⑤ ゴルフウィング勤務の時給**
`cad_partners.hourly_wage`。勤務は `cad_dispatches.kind='golfwing'`（`work_hours` 保持・client/sales なし）として登録し、`fee_amount = 時間×時給`。
partner_id を持つため既存 `refresh_caddy_finance` が**キャディ事業の外注費**として自動集計（ユーザー決定）。ゴルフウィングへの請求書は作らず、キャディ→YOZAN請求書に合算される。

**追加**: 委託先別の月次支払一覧＋**振込用CSV書き出し**、受取/支払とも `cad_invoices` へ発行記録（snapshot）→ **未入金・未払アラート**と入金消込。
`cad_invoices` に `kind`（receivable/payable）・`partner_id` を追加、`client_id` を nullable 化。
