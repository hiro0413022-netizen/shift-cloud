# AI DEMO SALES（WEB SALES COMMAND CENTER）— 正典

クリニック・動物病院向けホームページ制作の**営業デモ高速生成**システム。
独立アプリ `apps/demo-sales`（port 3009・権限 `use_demo_sales`|`view_hq`・スキーマ `dms_*`・migration 0048・DECISIONS #54）。

## 1. プロダクト思想（ここがブレたら作り直す）

一般的な営業管理システム**ではない**。中心価値は3つ:

1. 成約可能性の高い営業先を見つける
2. **営業先ごとのホームページ案を事前に作る**（←プロダクトの中心）
3. 完成イメージを見せた状態で営業する

「御院向けにホームページ案を作成しました。実際の画面を見て、必要かどうかをご判断ください」
という営業を、短時間・大量に実行できる状態がゴール。CRM・履歴・案件管理は補助機能。

- **現サイトを批判しない**: 「現在の魅力を活かしながら、スマホでより分かりやすく整理した案」として提案する。good_points（良い点）と caution_points（否定NGな点）を営業文書全体に差し込む設計
- **デモは仮**: 既存サイトの写真・文章・ロゴを無断利用しない。仮素材（※仮ラベル）で作り、契約後に正式素材へ差し替え
- **AI診断より先にHPを高速で作る**（ユーザー指示 2026-07-14）: デモ生成はルールベース（Claude API不要・即時）。分析・スコアはClaude/人が後から埋める

## 2. データ（dms_* / 0048）

| テーブル | 役割 |
|---|---|
| dms_prospects | 営業先。status（25段階ファネル）・analysis(jsonb)・score・good/improve/caution/sales_points・想定プラン/金額・次回アクション・失注理由 |
| dms_demos | 営業デモ。token（/d/[token] 非公開URL）・passcode・expires_on・brief(jsonb=生成入力＋修正指示履歴)・html（単一ファイル）・version |
| dms_documents | 提案書/電話トーク/訪問トーク/メール/お礼メール5種/見積書案（kind別・Markdown） |
| dms_activities | 営業履歴（call/visit/mail/meeting/edit_request/status/note）＋ prospect_id null の directive（自然言語の営業指示） |
| dms_plans | 料金プラン（basic 298,000/16,500・growth 498,000/29,800。管理はDB直接 or 画面追加は後続） |
| dms_projects | 成約→正式制作案件。handover(jsonb) に顧客情報・分析・デモ・要望・履歴・未確認事項を丸ごと引継ぎ（WEB DEVELOPMENT COMMAND CENTER の受け皿） |

初期シード: ユーザー提供の営業先13件（2026-07-14）＋プラン2種。

## 3. デモ生成の仕組み

- `src/lib/templates.ts` — **業種別テンプレート10種**（内科/歯科/整形外科/小児科/皮膚科/眼科/耳鼻科/美容/動物病院/接骨院）。配色・語彙（患者さま/飼い主さま）・ヒーロー表現・仮の診療内容/初診案内を業種ごとに変えてあり「使い回し感」を出さない
- `src/lib/render-demo.ts` — DemoBrief（院名・診療時間・住所・強み・色・院長メッセージ等）＋テンプレート → レスポンシブ単一HTML。スマホは下部固定バー（電話/予約/アクセス）。DEMOリボン・noindexをHTML内に埋込み
- 配信 `/d/[token]` — service_role＋トークン照合（#12/#23と同型）。X-Robots-Tag noindex・no-store・有効期限（既定60日）・任意パスコード
- 再生成 — 既存briefを引き継ぎフォームの変更分だけ上書き、versionを+1、修正指示は brief.instructions と dms_activities(edit_request) に履歴が残る（面談中のその場修正に対応）

### 3.1 写真（0049 / demo-assets バケット）

- 入れられる箇所: **ヘッダー（ヒーロー背景）**・**院内/診察風景ギャラリー（最大6枚・キャプション付き）**・**院長/スタッフ写真**。brief の `heroImage` / `gallery[]` / `directorImage`（公開URL）
- アップロード: `/p/[id]` のデモ生成フォーム内。署名URLでブラウザから直PUT（サーバーアクションのbody上限回避・Lesson OS #50と同型）。送信前にcanvasで長辺1600px・JPEG 0.85に縮小するのでスマホ写真をそのまま選んでよい
- バケット `demo-assets` は**公開**（デモは認証なしで配信されるため）。パスは `demos/{company}/{prospect}/{ランダム}` で推測不可。書き込みは service_role の署名URLのみ
- **素材ルール（厳守）**: 院から提供された写真・フリー素材のみ。**既存サイトの写真・ロゴの無断利用はしない**（#54）。未設定の箇所は従来どおりグラデーション＋絵文字のプレースホルダで成立する
- 基調色: 管理画面はカラーピッカー＋プリセット10色（`COLOR_PRESETS`）。HEX直接入力も可・「業種標準に戻す」で空に戻せる

## 4. 画面

- `/` 営業司令: 自然言語の営業指示欄（directiveとして保存→Claudeが次セッションで実行）・ファネルKPI・本日の営業活動・営業先一覧・追加フォーム
- `/p/[id]` 営業先詳細: 基本情報・分析/スコア・デモ生成/再生成・共有設定（パスコード/期限）・ドキュメント一括生成・活動記録・成約→移行
- `/p/[id]/compare` 現サイトvsデモ比較（PC/スマホ切替・面談用の話す順メモつき）
- `/d/[token]` デモ配信（公開・認証不要・トークンゲート）

## 5. 運用フロー

登録 → （分析・スコア: Claude/人） → デモ生成（即時） → ドキュメント一括生成 → 電話（10分面談の獲得が目的） → 面談（compare画面・その場で修正再生成） → 見積 → 成約 → dms_projects へ移行。
