# 営業サポートシステム 仕様書（v0.2 ドラフト）

- 仮の名前: **Sales OS**（他社に売るときの商品名はあとで決める）
- 最初に使う商品: **PGA NOTE**（担当: 株式会社ファイン 福原さん）
- 立ち位置: Genesis とは別の独立したシステム。将来つなげる（今回は境界だけ決める）
- 作成日: 2026-07-09 / 更新: 2026-07-09

---

## 0. このシステムを一言でいうと

**「営業マンが、今日やることが一目でわかって、そのまま動けるシステム」**

エクセル3つ（問い合わせ・導入先・DM送付）に散らばっている情報を1つにまとめ、
「誰に・いつ・何をするか」を自動でリスト表示する。商品ごとに営業のやり方を
設定画面で自由に変えられるので、PGA NOTE でも、将来の自社HP作成の営業でも同じシステムで使える。

決まっていること:

| 項目 | 決定 |
|---|---|
| 会社の分け方 | 最初から複数の会社で使える形（他社にも売れる） |
| 作り方 | 既存YOZANと同じ仕組み、ただし別システムとして新しく作る |
| 最初に入れるデータ | 問い合わせ・導入先・DM送付 の3ファイル全部 |
| 設定の自由度 | プログラムを触らず、画面から営業のやり方を追加・削除できる |

---

## 1. 一番見たい画面：ホーム（やることリスト）

ログインして最初に出るのは、営業マンの**「今日やること」リスト**。
これが本システムの中心。他の画面はここから枝分かれする。

### ホームに出るもの

**① 今日のやること（TODO）**
- 「〇〇スクールに電話（モニター検討中）」のように、やること＋相手＋一言メモで並ぶ
- 期日が来たもの／今日のものを上から表示
- チェックを付ければ完了。ワンタップで「電話した」「メールした」を記録

**② 期限が過ぎているもの（赤で警告）**
- フォローが遅れている案件を上に出して、取りこぼしを防ぐ

**③ 新しく来た問い合わせ**
- 自分の担当に割り振られた新規問い合わせ。ここから即対応できる

**④ もうすぐの予定・リマインド**
- モニター期間の終わり、DMの2回目・3回目の送付日、商談アポなど

**⑤ 今月の自分の数字（ミニ集計）**
- 対応中の件数、今月の商談数・導入数をカードで表示

> ポイント: 営業マンは朝ここを見れば、その日の動きが全部わかる。
> 探さない・迷わない・忘れない、を目指す。

---

## 2. 営業マンがやりたいこと（この順で使えるように）

営業の一日をなぞって、必要な操作をそろえる。

1. **朝** ホームで今日のやること・遅れを確認
2. **対応** 相手を開いて、電話/メール/訪問の結果を1行メモで記録（自由文でOK）
3. **次の約束** 「3日後に再連絡」など次にやることと日付をセット → 自動でTODOに戻る
4. **進み具合の更新** 「問い合わせ → 商談 → モニター → 導入」を1タップで進める
5. **新規追加** 問い合わせが来たらその場で登録（電話・メール・経路を選ぶだけ）
6. **アウトバウンド** DMやテレアポの相手・送った日・反応を記録
7. **振り返り** 自分やチームの数字、どの経路がよく決まるかを見る

### 営業マンが「見たい」もの
- 今日と今週やること／遅れているもの
- 相手ごとのこれまでのやりとり（履歴が時系列で全部見える）
- 案件が今どの段階にいるか（ボードでまとめて俯瞰）
- どの集客経路（DM・会報誌・紹介・facebook等）が成果につながっているか
- 自分の成績・チームの成績

## 3. 全体の組み立て（3つの階層）

```
会社（利用する会社）        例: 株式会社ファイン / 自社YOZAN / 将来の販売先
  └ 商品（プロジェクト）     例: PGA NOTE / WING NOTE / HP作成
        ├ 営業のやり方の設定  … 経路・進み方の段階・入力項目を商品ごとに決める
        ├ 相手（見込み客・お客さん）
        │     └ 連絡先 / やりとり履歴 / モニター / 導入
        └ 集客の施策（DM・テレアポ・セミナー）→ 問い合わせにつながる
```

- **会社**＝データを分ける単位（他社に売るときの単位）
- **商品（プロジェクト）**＝「商品ごとに管理」の単位。営業のやり方は商品ごとに別々にできる
- だから「PGA NOTEの営業」と「自社のHP作成の営業」を、同じシステムで別のやり方で回せる

## 4. 主な画面

| 画面 | 何をする所か |
|---|---|
| **ホーム（やることリスト）** | 今日やること・遅れ・新規問い合わせ・リマインド・自分の数字。※最初に出る画面 |
| **相手（見込み客・お客さん）** | 会社/担当者の情報、これまでのやりとり履歴、次の約束、進み具合 |
| **案件ボード** | 「問い合わせ→商談→モニター→導入」を列にして、付箋を動かす感覚で進捗管理 |
| **問い合わせ受付** | 新しい問い合わせを登録（経路・連絡先を選ぶだけ） |
| **集客の施策** | DM送付リスト、テレアポ、セミナーの相手と反応を管理。宛名印刷用の書き出しも |
| **導入先** | 契約済みのお客さん一覧（商品別）。問い合わせからワンタップで「導入」に昇格 |
| **成績・分析** | 経路別・段階別・担当別の集計グラフ |
| **設定** | 営業のやり方（経路・段階・入力項目・自動リマインド）を画面から編集。※管理者のみ |

## 5. 商品ごとにやり方を変えられる仕組み（横展開の要）

「商品が変わると営業方法も変わる」への対応。プログラムを触らず、設定画面だけで:

1. **進み方の段階**を自由に作れる（PGA NOTEは「問い合わせ→商談→モニター→導入」、HP作成なら「相談→デザイン→制作→確認→公開」など）
2. **集客の経路**を自由に足せる（DM・会報誌・紹介・facebook・Web検索…）
3. **入力項目**を商品ごとに足せる（PGA NOTEなら「モニターID」、HP作成なら「公開希望日」など）
4. **商品まるごとコピー**で、既存の設定をひな形にして新商品をすぐ立ち上げ
5. **自動リマインド**（「モニター開始したら〇日後にTODOを自動で作る」など）

> 自社のHP作成営業を始めるときは、新しい商品を作って段階と項目を設定するだけ。作り直し不要。

## 6. 今のエクセルの困りごと → どう解決するか

| 今の困りごと | 解決 |
|---|---|
| 問い合わせ・導入先・DMが別ファイルで、同じ相手の流れが追えない | 1つにまとめ、相手ごとに履歴が全部つながる |
| ステータスが自由文で、全体の見通しが立たない | 段階を決めてボードで一覧。予測もできる |
| 経路の書き方がバラバラ（16種類）で集計できない | 経路をリストから選ぶ形にして名寄せ |
| DM・テレアポ・セミナーの反応が問い合わせと結びつかない | 施策と問い合わせをつなげ、どの施策が効いたか見える |
| 福原さんに属人化、引き継ぎ・複数人・他商品に広げられない | 複数人・複数商品で使える形に |

## 7. エクセル3ファイルの取り込み

| エクセル | どうする |
|---|---|
| 問い合わせリスト（92件＋セミナー・交流会） | 「相手」と「案件」と「やりとり履歴」に分けて取り込み。経路は名寄せ |
| 導入顧客名簿（84件） | 「導入先」として取り込み。連絡先3件・商品(PN/WN)・経路も |
| DM送付リスト（約690件） | 「DM施策」と「送付相手」に取り込み。送付日・宛名印刷情報・多店舗も |

取り込み時に相談したい点:
- 経路の書き方をどうそろえるか（統一表を一緒に作る）
- 自由文のステータスを、どの段階に振り分けるか
- 問い合わせと導入先で同じ会社をどう1つにまとめるか（表記ゆれ）

## 8. 使う人と権限

- 会社をまたいでデータは見えない（会社ごとに完全に分離）
- 役割の例: オーナー（全部）／管理者（設定変更）／リーダー（全案件）／担当（自分の担当だけ）／閲覧のみ
- 商品ごとに担当を割り当て、見える範囲を調整

## 11. 作る順番（案）

1. **土台**: 会社・商品・設定（段階/経路/入力項目）の骨組み、ログイン、権限
2. **ホーム＆案件**: やることリスト、相手・履歴、案件ボード、問い合わせ受付 ← ここが本命
3. **取り込み**: エクセル3ファイルを移行
4. **集客の施策**: DM・テレアポ・セミナー、宛名印刷書き出し、経路別の成果
5. **モニター・導入**: モニター管理、導入への昇格、商品別集計
6. **横展開仕上げ**: 商品コピー、自動リマインド、権限の細かい調整
7. **将来**: Genesis連携

まず 1〜3 ができれば、福原さんが今のエクセルをやめて PGA NOTE の運用を始められる。
