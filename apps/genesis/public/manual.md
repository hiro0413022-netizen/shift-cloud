# RUNBOOK — YOZAN GENESIS の使い方（役員向け）

対象: 経営メンバー（古川さん・小川さんなど。PCに詳しくなくてOK）。
URL: https://yozan-genesis.vercel.app （スマホでも使えます。ホーム画面に登録しておくと楽です）

---

## 1. GENESISとは

会社全体（GOLF WING・FRUNK GOLF・KALLINOS・キャディ事業）の「今」をひとつの画面で見て、AIの報告を受け、経営の判断をする場所です。**入力作業の場所ではありません**（日々の入力はShift Cloudなど現場アプリが担当）。

## 2. ログインする

1. 上のURLを開く
2. Shift Cloud（シフトアプリ）と同じID・パスワードでログイン
3. 「アクセス権がありません」→ 本部閲覧権限（view_hq）が未付与です。古川さんへ

## 3. 毎日見るのは3つだけ

1. **経営ダッシュボード**（開いてすぐの画面）: 会社の総合スコアと警告。**「今日、判断すべきこと」のリストが最重要**
2. **承認待ち**: AIや現場からの承認依頼。内容を見て「承認 / 却下」を押す
3. **問い合わせ受信箱**: お客様からの問い合わせとAIの返信案。確認して送信を承認

## 4. 各タブでできること

| タブ | できること |
|------|-----------|
| 経営ダッシュボード | 会社の健康診断。スコア・売上・KPI・今日の判断リスト |
| CEO AI 司令室 | 日次レポートの生成と閲覧。KPIの手動更新もここ |
| 承認待ち | AI・現場からの承認依頼に答える |
| 問い合わせ受信箱 | 顧客問い合わせの確認とAI返信案の承認 |
| 社内連絡 | 役員→古川さんへの連絡ノート。書けば流れない。古川さんが対応済みにする |
| 財務（売上・利益） | 月次の売上・経費・利益。実績の入力もここ |
| 契約・法務 | 契約書の保管状況と期限アラート（登録はLegal OS） |
| 資料室 | 出店計画などの社内資料をアップロード・ダウンロード |
| システム相関図 | どのシステムが動いていて何と繋がっているか。障害の確認 |
| AI社員一覧 | 21体のAI社員が「何を見て何をするか」の定義 |
| 経営メモ（AIの記憶） | AIが覚えている経営情報 |
| 決定事項ログ | 過去の経営・設計判断の記録 |
| 出来事ログ | システムをまたいだ出来事の記録 |
| 開発状況 | システム開発の進捗 |
| 未来シミュレーション | 売上・会員数などの見通し |
| 外部連携 | 外部サービスとの接続状況 |
| システム台帳（ID/URL） | 全システムのURL・アカウント情報（要ページパスワード） |

## 5. 資料室の使い方

1. 「資料室」タブ →「アップロード」でPCやスマホのファイルを選ぶ（25MBまで）
2. 分類（例: 出店計画）を入れて【アップロード】
3. ダウンロードは各行の【⬇ DL】。社外への共有はファイルを渡すのではなく、この画面を見られる人を増やすのが基本

## 6. メンバーの権限を管理する（古川さん用）

権限は **Shift Cloud の管理画面**で一元管理しています。

1. https://shift-cloud-shift-cloud.vercel.app → 管理画面 →「スタッフ」
2. 対象スタッフを編集 → **ロール**を選ぶ → 保存
   - **役員（本部閲覧）**: GENESISを見られる（閲覧・承認）。※小川うらら用に作成済み
   - **本部**: GENESISに加えShift Cloudの管理全般
   - アカウント未発行の人は、編集画面で**初期パスワード**（8文字以上）を設定すると発行されます
3. 外すときも同じ画面でロールを変えるだけ

## 困ったとき

| 症状 | 対処 |
|------|------|
| ログインできない | ID・パスワードを確認（Shift Cloudと共通）。ダメなら古川さんへ（パスワード再設定できます） |
| スコアや数字がおかしい | 「CEO AI 司令室」で日次レポートを再生成 →「KPI更新」。直らなければ判断リストの警告を確認 |
| 資料のアップロードが失敗する | 25MB超の可能性。ファイルを分割 or 圧縮して再試行 |
| 画面が崩れている | 再読み込み。スマホで直らなければ画面名を古川さんへ |

## してはいけないこと

- **システム台帳のパスワードを社外の人に見せない・送らない**
- 承認は内容を読んでから（AIの提案は下書きです。最終判断は人）
