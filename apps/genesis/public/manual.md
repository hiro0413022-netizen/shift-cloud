# RUNBOOK — YOZAN GENESIS の使い方（役員向け）

対象: 経営メンバー（古川さん・小川さんなど。PCに詳しくなくてOK）。
URL: https://yozan-genesis.vercel.app （スマホでも使えます。ホーム画面に登録しておくと楽です）

---

## 1. GENESISとは

会社全体（GOLF WING・FRANK GOLF・KALLINOS・キャディ事業）の「今」をひとつの画面で見て、AIの報告を受け、経営の判断をする場所です。**入力作業の場所ではありません**（日々の入力はShift Cloudなど現場アプリが担当）。

GENESISでやることは、突き詰めると4つです。

1. **見る**（ダッシュボード・データに聞く）
2. **決める**（承認待ち・成果物レビュー・問い合わせ受信箱）
3. **指示する**（改善提案 → 実行指示 → スタッフへ連絡）
4. **記録が残る**（決定事項ログ・出来事ログ。あとから追える）

## 2. ログインする

1. 上のURLを開く
2. Shift Cloud（シフトアプリ）と同じID・パスワードでログイン
3. 「アクセス権がありません」→ 本部閲覧権限（view_hq）が未付与です。古川さんへ

## 3. 毎日見るのは4つだけ

1. **経営ダッシュボード**（開いてすぐの画面）: 会社の総合スコアと警告。**「今日、判断すべきこと」のリストが最重要**
2. **承認待ち**: AIや現場からの承認依頼。内容を見て「承認 / 却下」を押す
3. **成果物レビュー**: AI社員が毎朝つくった成果物（原稿・資料など）。中身を見て承認 / 却下
4. **問い合わせ受信箱**: お客様からの問い合わせとAIの返信案。確認して送信を承認

## 4. 各タブでできること

**見る**

| タブ | できること |
|------|-----------|
| 経営ダッシュボード | 会社の健康診断。スコア・売上・KPI・今日の判断リスト |
| CEO AI 司令室 | 日次レポートの生成と閲覧。KPIの手動更新もここ |
| データに聞く | 売上・会員・体験予約・勤怠・給与・経理・契約に**日本語で質問**（→ §5） |
| 財務（売上・利益） | 月次の売上・経費・利益。実績の入力もここ |
| 契約・法務 | 契約書の保管状況と期限アラート（登録はLegal OS） |
| 予約申込 | ビジター予約の申込状況の閲覧（確定・折り返し電話はReserve OS / シフトアプリの「やること」で） |
| 未来シミュレーション | 売上・会員数などの見通し |

**決める・指示する**

| タブ | できること |
|------|-----------|
| 改善提案 | AIが見つけた課題と打ち手。ここから工程に分けて配れる（→ §6） |
| 実行指示 | **Genesisから指示を出す唯一の入口**。スタッフの「やること」・AI社員へ届く |
| AI自動実行 | AIのアクションの実行状況。自動 / 自動+取消枠 / 承認必須の3階層（→ §7） |
| 承認待ち | AI・現場からの承認依頼に答える |
| 成果物レビュー | AI社員がつくった成果物の確認と承認/却下。**承認しても自動配信はされません** |
| 問い合わせ受信箱 | 顧客問い合わせの確認とAI返信案の承認 |
| 社内連絡 | 役員→古川さんへの連絡ノート。書けば流れない。古川さんが対応済みにする |
| スタッフへ連絡 | 書いて送ると公式LINEのスタッフグループへ配信＋記録（→ §8） |

**管理・記録**

| タブ | できること |
|------|-----------|
| 資料室 | 出店計画などの社内資料をアップロード・ダウンロード（→ §9） |
| アカウント管理 | スタッフの役割・停止・ログイン発行・追加（→ §10） |
| システム相関図 | どのシステムが動いていて何と繋がっているか。障害の確認 |
| AI社員一覧 | 21体のAI社員が「何を見て・何を判断し・何をするか」の定義 |
| 経営メモ（AIの記憶） | AIが覚えている経営情報 |
| 決定事項ログ | 過去の経営・設計判断の記録 |
| 出来事ログ | システムをまたいだ出来事の記録 |
| 開発状況 | システム開発の進捗 |
| 外部連携 | 外部サービスとの接続状況 |

## 5. 聞きたいことを聞く（データに聞く）

日次レポートは「AIから押し出される情報」なので、**聞きたいことを聞く場所**を別に用意しています。

1. 「データに聞く」タブを開く
2. 日本語でそのまま聞く（例:「今月のGOLF WINGの体験入会率は？」「先月より粗利が落ちた事業は？」）
3. 答えと一緒に**実際に使った検索条件と件数**が出ます。数字が怪しいと思ったらここを見てください

- 数字を計算しているのはデータベース本体で、**AIは検索の仕方を書いているだけ**です（AIが数字を作文することはありません）
- Genesisからは全社（全店舗・給与・経理・契約を含む）が見えます。現場アプリからは見える範囲が絞られます

## 6. AIに動いてもらう（提案 → 指示 → 成果物）

流れは一本道です。**途中で必ず人が判断します**。

1. **改善提案**: AIが課題と打ち手を出す（最優先 / 推奨 / 余力があれば）
2. 良いと思ったら【工程を配る】: 施策を**工程（誰が・何を・どの順で）**に分解 →「AIに工程を下書きさせる」も使えます
3. 担当を選んで発行 → **実行指示**へ。スタッフには「やることリスト」、AI社員には指示書が届く
4. AI社員が作った原稿・資料は **成果物レビュー**に溜まる → 中身を見て承認 / 却下
5. **承認＝配信ではありません。** お客様への配信・課金・外部送信は、承認のあと人が実行します

## 7. AI自動実行（どこまで勝手にやるか）

AIのアクションはリスクで3階層に分かれています。「AI自動実行」タブで状況を確認できます。

| モード | 意味 | 今この設定になっているもの |
|--------|------|-----------------|
| **自動**（緑） | そのまま実行 | 分析・集計、レポート生成、KPI再計算、下書き作成、社内通知、AI社員への内部指示、Issue起票・PR作成・テスト実行、発注候補の記録 |
| **自動＋取消枠**（黄） | 待ってから実行。その間【取消】を押せば止まる | **今は該当なし**（下記のとおり試運転中のため承認必須に寄せています） |
| **承認必須**（赤） | 承認するまで動かない | **スタッフへの指示配信・LINE一斉配信・SNS投稿（試運転中）**／本番デプロイ、本番DB変更、課金・支払い、顧客への重要連絡、個人情報、契約、採用・解雇、値上げ・料金変更、方針変更 |

- **いまは外部・スタッフ向けの送信が全部いったん止まります**（AIの精度を見るための試運転 / DECISIONS #63）。溜まったら「承認待ち」で承認/却下してください
- AIの提案の質に納得できたら、スタッフ連絡やSNS投稿を「自動＋取消枠」（数分待って自動で出る・その間は取消可）に緩められます。判断は古川さん
- 定義にないアクションは**安全側（承認必須）**に倒れます
- すべての実行は記録が残ります。失敗したものは赤字で理由が出ます
- 「テスト実行を入れる」は無害な動作確認用。押しても業務には影響しません

## 8. スタッフへ連絡する（公式LINE配信）

1. 「スタッフへ連絡」タブを開く
2. **1行目が件名**になります。本文を書く（例:「明日は10時開店です。オープン前の清掃をお願いします」）
3. 配信先グループを選ぶ（**YOZANとGOLF WINGで公式アカウントが別**です。選び間違いに注意）
4. 必要なら「スタッフアプリの『やること』にも出す」にチェック →【送る】

- 数分以内にLINEグループに届きます。送った連絡は画面下に記録が残ります
- **文字のみ**です（画像の配信は未対応）
- 個別LINEをやめてここに集約する運用です。「言った・言わない」を無くすのが目的

## 9. 資料室の使い方

1. 「資料室」タブ →「アップロード」でPCやスマホのファイルを選ぶ（25MBまで）
2. 分類（例: 出店計画）を入れて【アップロード】
3. ダウンロードは各行の【⬇ DL】。社外への共有はファイルを渡すのではなく、この画面を見られる人を増やすのが基本

## 10. メンバーの権限を管理する（古川さん用）

**GENESISの「アカウント管理」タブで完結します**（Shift Cloudの管理画面でも同じことができます。データは共通）。

1. 「アカウント管理」タブを開く（**manage_staff 権限が必要**。無い人は閲覧のみ）
2. できること:
   - **役割を変える** … 役割を変えると各システムのアクセスが**一括で**切り替わります（本部系: Genesis / Money / Legal / Caddy / Survey、受付、勤怠、Reserve OS、Lesson）
   - **ログイン発行** … アカウント未発行の人に初期パスワード（8文字以上）を設定して発行
   - **パスワード再発行** … 忘れた人はここで
   - **有効 / 停止の切替** … 退職者は停止（全システム一括で入れなくなります）
   - **＋ スタッフを追加** … 氏名・ログインID・初期パスワード・役割を入れて追加
3. 「⚠ ログイン未発行」の警告が出ている人は、権限はあってもログインできません。【ログイン発行】を押してください

> Legal / Money の細かい担当範囲（セグメント権限）は、各アプリ側で管理します。

## 困ったとき

| 症状 | 対処 |
|------|------|
| ログインできない | ID・パスワードを確認（Shift Cloudと共通）。「アカウント管理」で古川さんがパスワード再発行できます |
| スコアや数字がおかしい | 「CEO AI 司令室」で日次レポートを再生成 →「KPI更新」。それでも変なら「データに聞く」で直接聞いて突き合わせる |
| 「データに聞く」の答えが怪しい | 一緒に出ている検索条件と件数を確認。おかしければ質問を具体的に（期間・店舗を明記）して聞き直す |
| AIの実行が「待機」のまま進まない | 「AI自動実行」で【今すぐキューを回す】。承認待ちで止まっている場合は承認/却下を押す |
| スタッフへの連絡がLINEに届かない | 配信先グループが正しいか確認（YOZAN / GOLF WING）。「送信失敗」表示なら古川さんへ |
| 資料のアップロードが失敗する | 25MB超の可能性。ファイルを分割 or 圧縮して再試行 |
| 画面が崩れている | 再読み込み。スマホで直らなければ画面名を古川さんへ |

## してはいけないこと

- **システム台帳のパスワードを社外の人に見せない・送らない**
- 承認は内容を読んでから（AIの提案は下書きです。最終判断は人）
- 成果物を承認しただけで「配信済み」と思わない（配信は人が実行します）
- 退職者のアカウントを消さずに放置しない（「アカウント管理」で**停止**にする）
