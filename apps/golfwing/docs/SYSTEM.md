## 2. システムの目的

ゴルフショップ（ゴルフウィング）が、**ゴルフ用品（シャフト・グリップ・ヘッド・ボール・アパレル・工房用品）を仕入先メーカーへ発注する業務を効率化する**ための社内専用Webシステム。

### 解決する課題

| 課題 | 解決方法 |
|---|---|
| Excelや手書きで管理していた発注書を電子化 | 発注フォームによる一括入力・メール下書き自動生成 |
| 仕入先ごとに掛け率が異なり計算が煩雑 | `supplier_rules` / `product_suppliers` による自動掛け率計算 |
| 複数顧客の発注を1つのメールにまとめる必要性 | バッチコード（batch_code）による同一仕入先発注の束ね処理 |
| 発注後の入荷管理・残注追跡が煩雑 | 入荷登録・残注一覧・検品ステータス管理 |
| 仕入先ごとにCC先が異なる | `suppliers.cc_emails` カラムでCC設定を仕入先ごとに管理 |
| 同一商品を複数仕入先から調達する場合の掛け率管理 | `product_suppliers` テーブルによる商品×仕入先の多対多管理 |
| 発注書をExcelでダウンロードしたい | `/api/receipts/download` による Excel (.xlsx) 出力機能 |

## 3. 利用者

| 役割 | 認証方法 | 権限 |
|---|---|---|
| **ショップスタッフ**（主利用者） | ユーザー名・パスワード（DB認証） | 全機能利用可（デモ除く） |
| **管理者** (`is_admin=1`) | 同上 | バックアップ・リストア機能も利用可 |
| **デモユーザー** | `/demo-login`（パスワード不要） | 閲覧のみ（書き込みは403） |

### テナント構成

```
tenant_id=0  → デモテナント（毎日 JST 00:00 にCron自動リセット）
tenant_id=1  → ゴルフウィング本番テナント
tenant_id=N  → 将来の販売先テナント（SaaS拡張予定）
```

## 5. 今後の役割

### 短期（〜3ヶ月）
- 現行ゴルフウィング本番システムとして安定稼働
- マルチテナント基盤を活かした他ゴルフショップへの横展開（SaaS）

### 中期（〜1年）
- YOZAN Genesis統合：受注管理モジュールとの連携（顧客データ共通化）
- メール送信の内部化（SendGrid/Resend APIを利用したサーバーサイド送信）
- LINE・FAX発注自動化

### 長期（〜3年）
- 在庫管理機能追加
- 発注AIエージェント（需要予測・自動発注提案）
- 業界標準EDI対応

## 6. 技術スタック概要

| 層 | 技術 |
|---|---|
| **ランタイム** | Cloudflare Workers (Edge) |
| **フレームワーク** | Hono v4.12.12 |
| **DB** | Cloudflare D1 (SQLite互換) |
| **フロントエンド** | HTML + Vanilla JS + Bootstrap 5.3 + Font Awesome 6.4 |
| **ビルドツール** | Vite 6.3 + @hono/vite-build |
| **デプロイ** | Cloudflare Pages (wrangler 4.4) |
| **認証** | Cookie + HMAC-SHA256署名付きトークン（Web Crypto API） |
| **言語** | TypeScript（バックエンド） / JavaScript（フロントエンド） |
| **Excel出力** | 自前実装 `xlsxHelper.ts`（外部ライブラリなし） |
