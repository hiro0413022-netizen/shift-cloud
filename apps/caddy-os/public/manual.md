# RUNBOOK — キャディ派遣管理（Caddy OS）現場手順書

対象: 派遣の記録・請求書発行をする担当者。
URL: https://caddy-os-omega.vercel.app

---

## 1. ログインする

- Shift Cloudと同じID・パスワード
- 「アクセス権がありません」→ キャディ権限（use_caddy）が未付与。古川さんへ

## 2. 派遣を記録する（1派遣=1行）

1. 【派遣一覧】→ 一括入力グリッドで入力（Excelのように**Enterで次の行**へ。日付・取引先は前の行から引き継がれます）
2. **取引先を選ぶと売上単価が自動で入る**／**委託先を選ぶと委託料が自動で入る**
3. **社員（林さんなど）の派遣は委託料0のまま**にする — 人件費は給与側で計上されるため、ここに金額を入れると二重計上になります（入れようとしてもエラーになります）
4. 登録すると通し番号は日付順に自動で振り直されます（手で採番しない）

## 3. 請求書を発行する（月締め後）

1. 【請求書】→ 取引先を選ぶ → 対象月を選ぶ
2. 明細（キャディ業務料 × 日付 × 人工数 × 単価）・消費税・合計が自動計算されます
3. 内容を確認して**ブラウザの印刷 → PDF保存**（そのまま送付できます）
4. 締め日は取引先マスタの設定（月末締め/20日締め）に従って自動で切られます

## 4. 出勤可否カレンダー

- 【出勤可否】で委託先キャディごとに ○可 / △要相談 / ×不可 をクリックで登録（空欄=未回答）
- **委託先キャディはShift Cloudのシフト・勤怠には載せない**（社員ではないため。載せると給与＝二重計上になります）

## 5. マスタ管理

- 取引先（ゴルフ場）: 単価・締め日を登録。**単価改定したらマスタを先に直してから**派遣を入力
- 委託先キャディ: 委託料単価を登録。新しい委託先はまずマスタ登録（未登録のまま台帳に書かない）

## 6. この事業の注意（担当者が知っておくこと)

- 薄利事業です。**月商80万円を切ると赤字**（固定給が損益分岐点。ダッシュボードに常時表示）
- 売上・粗利は自動で本部の財務（fin_entries）へ集約されます — 手で財務側に入力しない

## 困ったとき

| 症状 | 対処 |
|------|------|
| 請求書の金額が台帳と合わない | 対象月の派遣行の「人工数・単価」を確認（1行の入力ミスがそのまま請求に出ます） |
| 社員に委託料を入れたい | 入れない（給与と二重計上）。交通費も給与側です |
| 委託先が選択肢に出ない | マスタ未登録。先に委託先マスタへ登録 |
| 誤入力を消したい | 行の削除は論理削除。消した理由をメモに残す |
