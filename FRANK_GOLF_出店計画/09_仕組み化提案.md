# YOZAN GENESIS：資料作成・業務計画の仕組み化 提案

株式会社YOZAN ／ 2026-07-09

## 思想（前提）
**「人が判断し、AIが調査・整理・資料化・シミュレーションを高速化する」。**
- **AIが自律的に進めてよい**：調査、分析、スケジュール作成、リスク洗い出し、Excelモデル作成、説明文・下書き作成、改善提案、資料化。
- **必ず人の承認が必要（AIは実行しない）**：契約、支払い、外部送信、法的判断、個人情報の利用、本番デプロイ/本番DB変更、採用/解雇、値上げ、大型支払い、顧客への重要連絡、方針変更。
- これはVISION（`docs/genesis/VISION.md`）のAI権限線引きと一致。

## 仕組みの全体像
新しい出店・投資・大型意思決定のたびに、以下を**テンプレートから生成**して回す：

```
案件フォルダ/
  00_計画サマリー（必要資料・不足情報・仮置き前提・成果物・順序）
  01_出店スケジュール（テンプレ→タスク13項目・2パターン・承認ポイント）
  02_ガントチャート.csv
  03_収支計画.xlsx（Excelモデル作成ルールに従い11シート・数式連動）
  04_承諾/提出資料（承諾依頼/金融機関/出資者テンプレから）
  05_リスク一覧（リスク管理テンプレ）
  06_KPI（事業別KPIテンプレ）
  DECISIONS.md（意思決定ログ・追記式）
  Assumptions.md（前提条件の一元管理・版管理）
  資料作成チェックリスト（納品前セルフ点検）
```

## 提供テンプレート（`templates/` に格納）
| ファイル | 用途 |
|---|---|
| `出店計画テンプレート.md` | 物件〜開業の工程を13項目で。最短/安全2パターン・承認ポイント付き |
| `収支計画テンプレート.md` | 11シート構成・入力/数式/シナリオの設計指針 |
| `承諾依頼資料テンプレート.md` | 関係先（ファイン等）への承諾依頼8章構成 |
| `金融機関提出資料テンプレート.md` | 融資審査向けサマリー構成 |
| `出資者説明資料テンプレート.md` | 出資者向けピッチ構成 |
| `事業別KPIテンプレート.md` | GOLF WING 5大KPI等の事業別KPI定義 |
| `リスク管理テンプレート.md` | 影響度×発生可能性×対策の一覧様式 |
| `資料作成チェックリスト.md` | 納品前の品質・数式・仮置き明記の点検 |
| `Excelモデル作成ルール.md` | 数式連動・色分け・シナリオ切替の必須ルール |
| `AI標準プロンプト集.md` | AIに資料を作らせる定型プロンプト |
| `DECISIONS.md` | 意思決定ログ（案件横断・追記式） |
| `Assumptions.md` | 前提条件の一元管理（版・根拠・保守補正） |

## 運用ルール（3点）
1. **仮置きは必ず明記**：確定前の数字・判断は「〔仮置き〕」と根拠を併記。実データ確定で上書き。
2. **前提はAssumptions一元管理**：Excelも文書も、数字はAssumptionsを唯一の源泉として参照。変更は1箇所で全体に波及。
3. **意思決定はDECISIONSに記録**：誰が・いつ・何を・なぜ決めたかを追記。後から追跡可能に。

## GENESISへの組込み（将来）
- 「資料AI（資料作成担当）」に本テンプレ群と標準プロンプトを紐付け、案件名を渡すと初版一式を自動生成。
- KPI（会員数・入会率・退会率・現金残高）と収支Excelを連携し、実績で前提を自動更新。
- ただし**承認が必要な行為はAIが実行せず、人の判断待ちキュー**に載せる（VISION準拠）。
