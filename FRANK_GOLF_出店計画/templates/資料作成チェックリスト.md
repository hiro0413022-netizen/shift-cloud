# 資料作成 チェックリスト（納品前セルフ点検）

## 共通
- [ ] 仮置きの数字・判断に「〔仮置き〕」＋根拠を明記したか
- [ ] 数字の源泉は Assumptions に一元化され、他は参照になっているか
- [ ] 第三者（ファイン/金融機関/出資者）が読んで実行可能性・採算性・リスクが分かるか
- [ ] 意思決定は DECISIONS.md に記録したか
- [ ] AIが承認必須の行為（契約/支払/外部送信/個人情報/法的判断）を勝手に実行していないか

## スケジュール
- [ ] 13項目（No/カテゴリ/タスク/詳細/開始/終了/所要/前後関係/担当/成果物/完了条件/遅延リスク/対策/優先度）を満たすか
- [ ] 最短・安全の2パターンがあるか
- [ ] 承認ポイント（誰が・何を判断すればGOか）が明記されているか
- [ ] マイルストーン・遅延時代替案があるか

## 収支Excel
- [ ] 11シート揃っているか
- [ ] 4シナリオ（楽観/標準/悲観/最悪）が切替で自動更新されるか
- [ ] 損益分岐点・月次CF・投資回収・資金ショート月が確認できるか
- [ ] 費用の抜け漏れ（修繕/税理士/予備費/保証金/予備費等）がないか
- [ ] 数式エラー0・色分け・保守前提になっているか

## 承諾/提出資料
- [ ] 目的・関係性・責任分担・費用負担・承認フローが明確か
- [ ] 確認事項に合意欄があるか
- [ ] リスクと対策が具体的か
