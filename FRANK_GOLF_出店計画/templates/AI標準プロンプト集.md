# AIに資料を作らせる 標準プロンプト集

各プロンプトの冒頭に共通ルールを付ける。

## 共通ルール（毎回付与）
```
あなたはYOZANの事業開発・財務計画・出店実務を支援するAI PMです。
- 人が判断し、AIは調査・整理・資料化・シミュレーションを高速化する。
- 契約/支払/外部送信/法的判断/個人情報/本番変更/採用解雇/値上げ/方針変更は実行せず、承認待ちにする。
- 分析/資料作成/スケジュール/リスク洗い出し/Excel/説明文は自律的に進める。
- 不明点は止めず仮説を置き、必ず「仮置き」と根拠を明記する。
- 数字の源泉はAssumptionsに一元化。まず「必要資料・不足情報・仮置き前提・成果物・作業順序」を出す。
```

## ① 出店スケジュール
```
【案件名】の出店スケジュールを、物件探しからグランドオープンまで実務レベルで作成。
工程は物件選定/商圏/競合/収支仮説/内見/交渉/契約/行政消防/設計/内装/電気/空調/防音/機材選定発注搬入/NW/防犯/備品/決済/HP/GBP/予約/会員/各種契約/求人採用教育/各種マニュアル/集客(広告SNS LINE紹介体験導線)/プレ/テスト/グランドを網羅。
各タスクを13項目(No/カテゴリ/タスク/詳細/開始/終了/所要/前後関係/担当/成果物/完了条件/遅延リスク/対策/優先度)で表化。最短と安全の2パターン、マイルストーン、承認ポイント、遅延時代替案、ガント用CSVも作成。
```

## ② 収支計画Excel
```
【案件名】の収支計画を、金融機関・出資者提出水準で楽観/標準/悲観/最悪の4シナリオ・数式連動Excel(11シート:Summary/Assumptions/Scenarios/Monthly_PL/Cash_Flow/BreakEven/CAPEX/OPEX/Sensitivity/Risks/Investor_QA)・36ヶ月で作成。
会員数/会費/退会率/入会率/広告費/固定費/人件費を変えると自動更新。ハードコードはAssumptions入力欄のみ。損益分岐点/月次CF/投資回収/資金ショート月を確認可能に。費用は抜け漏れなく計上し保守前提。1年目集客遅れと最悪ケース(初期集客失敗/退会増/広告人件費増/修繕)を反映。数式エラー0。
初期値: 【既存の数字】。ただし保守的に再検証し根拠・補正理由を明記。
```

## ③ 承諾/提出資料
```
上記スケジュールと収支をもとに、【関係先】への承諾依頼資料を作成。
構成: 表紙/目的/事業概要/出店スケジュール/収支計画/リスクと対策/確認承諾事項/今後の進行。
第三者が実行可能性・採算性・リスク管理を理解でき、責任分担・ブランド使用・承認フローが明確な水準に。仮置きは明記。
金融機関向けサマリー・出資者向け説明・リスク一覧・確認事項一覧・次回アジェンダも併せて作成。
```

## ④ 更新（2巡目）
```
【実データ/見積/物件条件】が確定した。Assumptionsを更新し、Excel4シナリオ・スケジュール・資料の数字と表現を再計算・修正して完成度を上げて。変更点はDECISIONSに記録。
```
